Definición de la función de costes¶

In [38]:
def compute_cost(theta, x, y):
    m = len(y)
    y_pred = logistic_function(np.dot(x , theta))
    error = (y * np.log(y_pred)) + ((1 - y) * np.log(1 - y_pred))
    cost = -1 / m * sum(error)
    gradient = 1 / m * np.dot(x.transpose(), (y_pred - y))
    return cost[0] , gradient

def gradient_descent(x, y, theta, alpha, iterations):
    costs = []
    for i in range(iterations):
        cost, gradient = compute_cost(theta, x, y)
        theta -= (alpha * gradient)
        costs.append(cost)
    return theta, costs

Esto demuestra que el riesgo de aumentar el valor del ratio de aprendizaje para una coste de computacin menor no tiene sentido, ya que la diferencia entre 10 iteracciones y 200 iteracciones no vara mucho el tiempo de carga para el ordenador, por lo que es mejor buscar un valor medio con un mayor nmero de iteracciones para as no arriesgarse a tener un error en la bsqueda del mnimo coste de la funcin.
